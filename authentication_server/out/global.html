(async) authenticate(walletPublicKey, challenge, signature, relyingPartyId)

This route is called by the client application to answer to an asymmetric challenge-response; the request is a JSON object containing the following parameters:
Parameters:
Name Type Description
walletPublicKey user's wallet public-key (RSA public-key in pem format, then base64 encoded)
challenge random nonce previously received by the user to be signed in order to prove passkey's possession
signature signature computed using passkey's secret-key to be validated by the server
relyingPartyId relying party's unique identifier, in order to authenticate correct user to a specific service

Parameters:
Name Type Description

relyingPartyId relying party's unique identifier

username user's identifier within relying party's provided service/application
passkeyPublicKey passkey's public-key associated to relying party's auhtentication (RSA public-key in pem format, then base64 encoded)
passkeySecretKeyE2E passkey's secret-key associated to relying party's authentication (RSA secret-key in pem format, AES-256-CBC encrypted, then base64 encoded)
